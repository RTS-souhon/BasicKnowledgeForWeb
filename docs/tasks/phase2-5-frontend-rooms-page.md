# Phase 2-5 Frontend: `/rooms`

## 対象

- `docs/pages/06-rooms.md`

## 目的

部屋割り一覧を閲覧できるページを実装する。  
Phase 2 では閲覧専用とし、編集 UI は入れない。

## 実装内容

1. Server Component として `/rooms` を実装する
2. backend の `GET /api/rooms` からデータ取得する
3. デスクトップでは表形式、スマートフォンではカード形式にする
4. 空状態の表示を用意する

## Phase 2 でやらないこと

- 追加 / 編集 / 削除 UI
- admin 向け編集ボタン

## テスト

- 一覧が表示される
- room 名順に表示される
- モバイル想定でカードレイアウトが崩れない
- 空状態が表示される

## 完了条件

- `/rooms` が認証後に閲覧できる
- page test が追加されている
