# Phase 2-7 Frontend: `/shop`

## 対象

- `docs/pages/08-shop.md`

## 目的

販売物一覧を閲覧できるページを実装する。  
Phase 2 では閲覧専用とし、編集 UI は入れない。

## 実装内容

1. Server Component として `/shop` を実装する
2. backend の `GET /api/shop-items` からデータ取得する
3. すべてのアイテムに対して Cloudflare R2 由来の `image_url`（API レスポンスの必須フィールド）を表示する
4. 価格表示と在庫ステータス表示を整える
5. スマートフォンではカード形式にする（カード上部に画像を表示）
6. `stock_status` ごとのバッジ文言と色を統一する
7. 空状態の表示を用意する（画像無し環境でもレイアウトが崩れないようにプレースホルダーを用意）

### 画像表示要件

- `/shop` の一覧はテーブル（デスクトップ） / カード（モバイル）いずれも必ず商品画像を表示する。
- 画像は `https://assets.reitaisai.info/<image_key>` のような公開 URL を `image_url` として受け取り、`<img>` / `next/image` で描画する。
- サイズ: デスクトップは 100px のサムネイル（object-fit: cover）、モバイルはカード幅いっぱい 4:3。
- alt テキストは `name` を使用し、画像読み込み失敗時はグレーのプレースホルダーを表示する。

## Phase 2 でやらないこと

- 追加 / 編集 / 削除 UI
- admin / developer 向け編集ボタン

## テスト

- 一覧が表示される
- 在庫ステータスの文言と見た目が正しい
- name 順に表示される
- 空状態が表示される
- 各アイテムの画像が表示され、`image_url` なしの場合はエラーメッセージ（`データ不備`）を表示する

## 完了条件

- `/shop` が認証後に閲覧できる
- page test が追加されている
