# Phase 3-2 Frontend: `/search`

## 対象

- `docs/implementation-plan.md`
- `docs/pages/09-search.md`

## 目的

横断検索ページを実装し、ユーザーがキーワード入力のみで現在の会期データ（タイムテーブル / 部屋割り / 企画 / 販売物 / その他）を素早く探せるようにする。

## 実装内容

1. `(authenticated)/search/page.tsx` を Client Component として作成し、URL クエリ `?q=` で検索状態を管理する
2. `fetch` で `GET /api/search?q=<keyword> (header: x-event-id)` を呼び出し、会期未選択時は検索 UI のみ表示する
3. 入力欄とボタンをモバイルではフルワイド縦配置、デスクトップでは横並びにする
4. 各カテゴリごとに結果セクションを作り、件数とリスト（カード表示）を描画する
5. 全カテゴリ 0 件のときは「該当する情報が見つかりません」を表示する

## このフェーズでやらないこと

- 編集 UI や管理者向け操作（Phase 4 で対応）
- バックエンドの検索 API 実装（Phase 3-1 で対応）

## テスト

- 検索フィールドとボタンが描画される
- クエリが存在するとき `GET /api/search` を呼び、カテゴリ別結果が表示される
- 該当カテゴリがない場合はそのセクションを描画しない
- 0 件時に空状態メッセージが表示される
- 空文字で検索するとバリデーションエラー（またはボタン無効）が発生する

## 完了条件

- `/search` ページが `(authenticated)` レイアウトで利用でき、Page テストが追加されている
