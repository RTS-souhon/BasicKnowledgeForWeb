# ユーザー登録画面 `/register`

## 概要

名前・メールアドレス・パスワードでアカウントを作成する。**実装済み**。

---

## アクセス制御

| 条件 | 挙動 |
|---|---|
| 誰でもアクセス可 | 表示 |

---

## 画面構成（実装済み）

単一カラムのフォームのため、デスクトップ・スマートフォンで共通のレイアウトを使用する。

```
┌─────────────────────────────┐
│        ユーザー登録          │
│  名前           [          ] │
│  メールアドレス  [          ] │
│  パスワード      [          ] │
│  パスワード（確認）[         ] │
│  [エラーメッセージ（任意）]   │
│         [登録する]            │
└─────────────────────────────┘
```

**スマートフォン固有の要件:**
- 入力欄・ボタンはフルワイド（`w-full`）
- メールフィールドに `inputMode="email"`、名前フィールドに `autoComplete="name"` を付与

---

## フォームバリデーション

バックエンドの `createUserSchema`（`apps/backend/src/infrastructure/validators/userValidator.ts`）を拡張して使用。

| フィールド | ルール |
|---|---|
| name | 必須・255文字以内 |
| email | 必須・メール形式・255文字以内 |
| password | 必須・8文字以上 |
| confirmPassword | password と一致 |

---

## API

### `POST /api/users`

| ステータス | 内容 |
|---|---|
| 201 | `{ "user": { id, name, email, role } }` |
| 400 | `{ "error": "このメールアドレスは既に使用されています" }` |
| 400 | `{ "error": "バリデーションエラー", "details": [...] }` |
| 500 | `{ "error": "ユーザーの作成に失敗しました" }` |

---

## 実装メモ

- 登録後は自動ログインせず、成功メッセージを表示してフォームを切り替える
- 新規登録ユーザーの `role` は常に `'user'`（admin/developer への昇格は admin が別途行う）

---

## テスト項目（実装済み）

| # | テスト内容 |
|---|---|
| 1 | フォームフィールドが表示されること |
| 2 | 空送信でバリデーションエラーが表示されること |
| 3 | パスワード不一致でエラーが表示されること |
| 4 | 正常登録で成功メッセージが表示されること |
| 5 | メール重複でメールフィールドにエラーが表示されること |
| 6 | サーバーエラーでグローバルエラーが表示されること |
